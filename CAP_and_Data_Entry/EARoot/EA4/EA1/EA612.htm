The CAP result data is intended to be a temporary store for assay result data points.<br /></p><p><br /></p><p>Many of these states are set by automated processes.  Only 'Approved' and 'Rejected' are set by researchers.<br /></p><p><br /></p><p>The cycle is determined by the upload cycle (say, daily).  At the start of an upload (extract) the status is set to extraction started.  At the completion, the status is further updated to extraction completed.<br /></p><p>In the next cycle the status is updated from 'Rejected'/'Approved' to 'Mark for Deletion'.<br /></p><p>In the final cycle 'Mark for Deletion' results are deleted for the database.<br /></p><p><br /></p><p>This gives data generators 2 days after approving results before they are deleted. After deletion the results are only available through the OLAP and query tools.<br /></p><p>In the initial year of operation deletion will not be used so as to assist trouble-shooting<br /></p>
